2026-05-18 ⚠️ 发现增强泄漏 · 切换到 StratifiedGroupKFold critical fix
commit bf9f89b · "docs: record grouped validation experiments"

数据中每个 lesion 有 3 个版本：100.jpg / 100_aug1.jpg / 100_aug2.jpg。随机按图片划分会让训练集和验证集出现同一个 lesion 的不同版本，导致模型只是在认增强。

修正方案：StratifiedGroupKFold，group key 取 base_id_from_image_id（去掉 _aug* 后缀），保证一个 lesion 的所有版本永远在同一 fold。

诚实分数立刻下跌：

协议image-level（错）grouped CV（正确）
Macro-F1（早期 all_boundary+LR）~0.910.7454

这是整个项目最大的方法学贡献，也是助教/老师最看重的一点。

2026-05-28 分支 tta-nested-fusion：验证 0.8055 是不是真的 negative branch · not merged
commit 55da6bd · experiments/run_tta_nested_fusion.py · 676 行

怀疑：现有 fusion 在同一组 OOF 行上既训分类器又调 weight，是 in-sample 过拟合。方案：嵌套 CV（外 5 fold + 内 3 fold 选 weight）+ 6 变换 TTA 平均（identity / hflip / vflip / rot90 / rot180 / rot270）。

4-cell ablation，5-seed bagged：

cellTTAtune wBal-AccMacro-F1Accuracy
A否否 (w=0.5)0.77630.76070.7417
B是否 (w=0.5)0.77530.76100.7433
C否是 (nested)0.77150.75710.7383
D是是 (nested)0.77260.75840.7400

Cell A 逐位复现现有 fusion bagged（pipeline sanity 通过）。Cells B/C/D 全部 |Δ| < 0.005。Nested-w 诚实地选 w = 0.562 ± 0.143，不是 0.5。

结论：0.8055 是 in-sample 噪声，0.7763 才是 late fusion 的诚实天花板。

5 · 探索闭环 · 完整叙事

故事弧

  1. 开局陷阱：image-level 划分跑出 ~90% F1，看上去很好。
  2. 方法学修正：识别增强泄漏 → 切到 StratifiedGroupKFold by base_id，分数诚实回落到 70-78 区间。
  3. 主线选定：系统化 grid search 后，all_abcd_grouped + LR03 + k=140 在 5-seed 稳定性下胜出，定为官方传统主线。
  4. 探索性候选：XGBoost cascade、early fusion、late fusion 三条独立路线全部跑过；其中 late fusion 单 seed 见到 0.8055 BalAcc，但 5-seed bagged 回落到 0.7763，悬而未决。
  5. 怀疑 → 验证：分支 tta-nested-fusion 用 TTA + 嵌套 CV 选权重做诚实复测，4-cell ablation 显示 0.8055 是 in-sample noise，fusion 的真上限就是 0.7763。
  6. 错误归因：分析主线 144 个错误中 25 个"全错 lesion"，分桶发现 15+ 是真实的 mel/nv 表型重叠（不可救），仅 2 个属于毛发遮挡（疑似可救）。
  7. 最后一击：分支 hair-removal-eval 跑 DullRazor 预处理 → bagged BalAcc -0.022，闭合最后一个"算法可救"的桶。
  8. 闭环：0.78 BalAcc 从"经验上限"升级为"有证据的结构性上限"。剩余误差归因为生物学表型重叠，需要 dermoscopic structural features 或 DL 才能进一步突破 → 自然引出 MobileNetV2 0.8533 BalAcc 作为对照扩展。

一句话总结：我们做了完整的方法学修正、系统化模型搜索、嵌套 CV 验证、错误归因和针对性预处理实验。诚实的传统 ML 上限是 5-seed bagged BalAcc 0.78，由生物学表型决定；深度学习扩展可达 0.85，但按课程规则不进入主提交。
