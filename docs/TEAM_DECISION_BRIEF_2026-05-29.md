## 一、我们做了什么（工作全貌）

### 1.1 方法学核心贡献：防止增强泄漏

数据是 200 张原图 + 400 张增强图（每张原图 2 个增强版），共 600 张。**早期我们用 image-level 随机切分时拿到过 90%+ 的虚高分**，因为同一个 lesion 的原图和增强版会同时出现在训练集和验证集里，模型其实只是在"认增强版本"。

修正方法：用 `StratifiedGroupKFold`，按 `base_id`（lesion 唯一标识）分组，**保证同一 lesion 的原图和所有增强版本都在同一折**。修正后所有结果掉到 70-80% 区间——这才是真实的模型能力。

> **这是项目最核心的方法学贡献**。任何只看分数高低、不区分协议的对比都是无效对比。

### 1.2 特征工程全图（5 大类，13 个模块）

我们做了 13 个特征模块，覆盖临床 ABCD 准则的所有维度：

| 类别 | 模块 | 含义 | 是否进主特征集 |
|---|---|---|---|
| **A 不对称（Asymmetry）**| `features_abcd_grouped`、`features_subregion` | 主轴/副轴方差比、子区域颜色差 | ✅（grouped 版进主）|
| **B 边界（Border）**| `features_boundary`（王昱航）| 边界凸包、分形维度、复杂度 | ✅ |
| **C 颜色（Color）**| `features_color`、`features_dermoscopy` | RGB/HSV/Lab 统计、皮肤镜结构色 | ✅ 颜色 / ❌ 皮肤镜结构 |
| **D 直径与形状（Dimension）**| `features_shape` | 面积、周长、圆度 | ✅ |
| **纹理（额外）**| `features_texture`、`features_lbp_multi`、`features_gabor` | LBP / GLCM / Gabor | 部分 |
| **病变-背景对比**| `features_contrast` | lesion 内外 Lab 差 | ✅ |
| **临床直觉特征**| `features_melnv` | 针对 mel/nv 易混的诊断特征 | ❌（负结果）|

主模型最终用的是 `all_abcd_grouped`（252 维），经过 `SelectKBest(k=140)` 筛选。

### 1.3 模型探索全图（4 条主线）

| 主线 | 方法 | 目的 |
|---|---|---|
| **L1 主线性模型** | LR(C=0.3) + class_weight=balanced + k=140 | 主提交候选；强可解释、强稳定 |
| **L2 SVM-RBF**（章蓉）| ABCD + SVM-RBF + mel 阈值调整 | 验证非线性模型在小样本上的天花板 |
| **L3 XGBoost 级联** | 2 阶段：vasc-vs-rest → mel-vs-nv | 利用类不平衡的结构先验（vasc 仅 90/600）|
| **L4 深度学习扩展**（刘智瑞）| MobileNetV2 + ImageNet 预训练 + crop + TTA | 现代方法对照；不进定量主提交 |

### 1.4 融合实验全图（3 轮全部失败）

| 融合方式 | 设计动机 | 5 种子 bagged 结果 | 结论 |
|---|---|---|---|
| **早期融合**（特征拼接）| 把所有传统特征拼成一个超长向量 | BalAcc 0.7766 | 单种子能到 0.7997，bagged 不稳 |
| **后期融合 0.5/0.5**（LR + 级联）| 概率平均 | BalAcc 0.7763 | 单种子 0.8055（高分），bagged 比纯级联低 |
| **TTA + 嵌套 CV 融合**（5/28）| 用嵌套 CV 学融合权重 + TTA | BalAcc 0.7726 | 比 0.5/0.5 还差 |
| **逐类融合 Path A**（5/29）| 每类一个权重，1331 组合网格 | seed 127 nested 0.7910 | w_mel std=0.258，权重不稳 |
| **结构 2 轴融合 Path A'**（5/29）| 把概率分解成 P(vasc) 和 P(mel\|非vasc) 两轴 | seed 127 nested 0.7907 | w2 std=0.244，仍然不稳 |

> **结论**：n=600 + vasc 仅 90 张，融合权重学习的方差太大，**任何"让数据自己学权重"的方案都比固定 0.5/0.5 差**。这本身是一个值得讲的发现。

## 三、模型选择讨论（开放）

### 3.1 三个候选的完整档案

#### 候选 A：主 LR

```text
配置：all_abcd_grouped + LogisticRegression(C=0.3, balanced) + SelectKBest(k=140)
5 种子 bagged：BalAcc 0.7512 / Macro-F1 0.7435 / Acc 0.7283
seed 127：BalAcc 0.7871 / Macro-F1 0.7715 / Acc 0.7600
per-seed std：BalAcc 0.0297, Macro-F1 0.0271
错误数（seed 127）：144 / 600
```

**优点**：
- 所有文档（`README.md`、`PROJECT_REVIEW_GUIDE.md`、`TEAM_TASKS.md`）都已锁定，不需要改口径
- LR 极易解释——可以输出每个特征的权重，对应到 ABCD 临床意义上
- 单一模型，不需要 bagging 部署，直接 joblib 加载预测
- 当前 `outputs/models/ml_baseline.joblib` 已就绪
- 风险最低：评估方法、复现方法、教学叙事全部已经走通

**缺点**：
- 5 种子 bagged BalAcc 是所有候选里**最低的**（0.7512）
- 老师如果按 BalAcc 评分，这个候选直接吃亏 3-4 个百分点

#### 候选 B：XGBoost 级联

```text
配置：Stage1 (xgb_cascade_stage1, k=100, vasc-vs-rest)
     Stage2 (xgb_cascade_stage2, k=120, mel-vs-nv | non-vasc)
     soft cascade: P(mel) = (1-P_vasc) * stage2_mel
5 种子 bagged：BalAcc 0.7839 / Macro-F1 0.7609 / Acc 0.7450
seed 127：BalAcc 0.7941 / Macro-F1 0.7748 / Acc 0.7567
per-seed std：0.0206
错误数（seed 127）：146 / 600
```

**优点**：
- 5 种子 bagged BalAcc **比主 LR 高 +0.0327**（约 1.5σ，统计显著）
- 利用了 vasc 类（90 张）的结构先验——把极度不平衡的多分类降为两个 2 分类问题
- 已经过 5/28-29 全方位压力测试：所有"再加点东西"的尝试（A1/A2/A3/B1/dermoscopy/hair removal/tta-nested）**全部失败**，说明它已经在饱和点附近
- 完美呼应"我们利用了医学问题的结构"这一叙事

**缺点**：
- per-seed std 0.0206（是 LR 的 1.5 倍）——单种子下限可能掉到 0.74
- `(1-P_vasc) * stage2_mel` 的耦合意味着 Stage 1 一旦在测试集上对某个非 vasc 样本过度自信，那条样本的 mel/nv 概率会塌陷
- 需要写 5 种子 bagged 训练器和 `run.py cascade_bagged` 模式（约 2 小时工作量）
- README 和现有口径要改：从"主 LR"切换到"主级联，LR 是基线"

#### 候选 C：后期融合 0.5/0.5（LR + 级联）

```text
配置：P_final = 0.5 * P_LR + 0.5 * P_cascade
5 种子 bagged：BalAcc 0.7763 / Macro-F1 0.7607 / Acc 0.7417
seed 127：BalAcc 0.8055（项目最高分单种子结果）
```

**优点**：
- 单种子最高分 0.8055 是项目里最亮眼的数字
- 概念上"组合两个模型的优点"很好讲

**缺点**：
- 5 种子 bagged 0.7763 反而**比纯级联低**——单种子高分是过拟合 seed 127
- 我们已经做过 4 种融合权重学习（flat / per-class / structural / nested）全部失败
- 部署最复杂：需要同时维护 LR + 级联两个模型，出问题点最多
- 0.8055 写进报告会被严肃读者识破——这是单种子结果，不能当作稳定指标

### 3.2 选择维度（讨论时考虑这些）

| 维度 | 候选 A（主 LR）| 候选 B（级联）| 候选 C（融合）|
|---|---|---|---|
| **5 种子 bagged 分数** | 最低（0.7512）| **最高（0.7839）**| 中（0.7763）|
| **单种子最高分** | 0.7871 | 0.7941 | **0.8055** |
| **可解释性** | **极强**（LR 权重 → ABCD）| 中（树模型可看 importance）| 弱（两个模型组合）|
| **稳定性 / 可重现性** | **最好** | 较好（per-seed std 0.0206）| 较差 |
| **部署复杂度** | **最低** | 中（需要 5 种子 bagged）| 高 |
| **报告口径切换成本** | **零**（README 已锁）| 中（要改 README）| 中 |
| **答辩可讲性** | "稳定可解释"路线 | "结构先验红利"路线 | "1+1>1"但要老实说不稳 |
| **风险** | **最低**（评估老熟）| 测试分布漂移会放大级联耦合风险 | 部署最容易出 bug |

### 3.3 决策应该问自己的问题

1. **老师是按什么评分？**
   - 如果是 BalAcc 单一指标 → 候选 B
   - 如果看可解释性 + 工程严谨度 → 候选 A
   - 如果看综合（含口头汇报）→ 任意一个都行，看叙事

2. **我们对级联在测试集上的 worst-case 有多担心？**
   - 训练集 vasc 占 15%，测试集如果分布相同，级联稳得住
   - 如果测试集 vasc 异常多/少，Stage 1 阈值会失调

3. **我们想讲一个什么样的故事？**
   - "我们做了一个稳定可解释的传统 ML 模型" → A
   - "我们利用了医学问题的结构先验" → B
   - "我们尝试了所有方法，最稳的赢" → A 或 B（任选）

4. **5 种子 bagged 训练器的开发成本（约 2 小时 + 验证）我们承担得起吗？**
   - 承担得起 → B 是真选项
   - 承担不起 → 只能 A

## 四、叙事方案讨论（开放）

下面 4 种叙事框架是基于现有结果**真实成立的**版本。挑一个或者混搭。

### 叙事 A：保守路线（"稳"字优先）

**主线**：
> "我们做了一个严格 grouped CV 防泄漏的可解释传统 ML 主模型，在 5 种子下稳定达到 BalAcc 0.7512（seed 127 = 0.7871）。我们做了大量探索性实验（级联、融合、深度），但最终选择最稳定、最可解释的 LR 作为主提交。"

- **配模型**：候选 A
- **章节结构**：
  1. 数据 + 防泄漏（grouped CV，最重要的方法学贡献）
  2. 特征工程（13 模块全图）
  3. 主模型选择与稳定性
  4. 鲁棒性 + 错误分析
  5. 探索性扩展（级联、融合 = 单种子高分但不稳）
  6. 深度学习对照（MobileNetV2）
- **强项**：方法学严谨、分数老实
- **弱项**：看起来"分数不亮"（最高 5 种子 bagged 0.7512 在传统 ML 里中等）

### 叙事 B：进取路线（"利用结构先验"）

**主线**：
> "三类 mel/nv/vasc 的样本分布极度不平衡（90/210/300）。我们设计了两阶段 XGBoost 级联，先识别 vasc，再处理剩余两类，把多分类问题降为两个 2 分类问题。结果：5 种子 bagged BalAcc 0.7839，比线性基线 +0.0327。我们也尝试了融合主线性模型和级联，但所有权重学习方案都因数据规模 + 类不平衡而失败——这是融合在 n=600 上的天花板。"

- **配模型**：候选 B
- **章节结构**：
  1. 数据 + 防泄漏
  2. 关键观察：极度类不平衡需要利用结构先验
  3. 两阶段级联设计 + 各阶段错误来源分析（Stage 1 ~22% / Stage 2 ~78%）
  4. 5 种子 bagged 结果 + per-class 错误重叠分析
  5. 我们尝试过的所有"再加点东西"为什么失败（A1/A2/A3/B1/dermoscopy/hair removal/tta-nested）
  6. 融合实验：4 轮全部失败 + 数据规模 vs 权重学习方差的关系
  7. 鲁棒性 + 错误分析
  8. 深度学习对照
- **强项**：分数亮、有理论依据、负结果丰富
- **弱项**：得说服老师"级联不是黑箱"——需要画图解释 P(vasc) 软门的几何含义

### 叙事 C：方法学优先（"实验严谨度"）

**主线**：
> "皮肤病变分类的真正挑战不是模型选择，而是评估协议。早期我们用 image-level CV 拿到过 90%+ 的虚高分，发现是增强泄漏后切换到 grouped CV，所有结果都掉到 70-80% 区间。这才是真实的模型能力。我们用严格协议下做了 16 大类实验，包括 4 种融合策略、4 种医学预处理、22 维皮肤镜临床特征、TTA + 嵌套 CV，全部诚实报告。"

- **配模型**：候选 A 或 B 都行
- **章节结构**：
  1. **核心方法学发现**：image-level → group-level，从 90%+ → 70%+
  2. 完整实验账本（16 节）总览
  3. 主模型 + 探索性候选
  4. 负结果驱动的边界判断（我们知道在 n=600 上什么是不可能的）
  5. 鲁棒性 + 错误分析 + 深度学习对照
- **强项**：成熟度最高、最难被挑刺
- **弱项**：分数不一定最亮、需要老师能 appreciate "正确做事比刷分更重要"

### 叙事 D：负结果驱动（"我们尝试了什么、为什么不用"）

**主线**：
> "我们做了 16 大类实验，10 个失败，6 个成功。失败的部分让我们知道在这个数据规模下什么是不可能做到的：融合权重学不出来、医学预处理在强级联上无效、皮肤镜特征会和 ABCD 冗余、去毛发因为只有 2 张样本影响 lesion 而无效。这些边界判断比简单刷分更有价值。最终主提交是 [级联 / LR]。"

- **配模型**：B 配比 A 好（B 的探索更彻底）
- **章节结构**：
  1. 数据 + 防泄漏
  2. 主模型（直接放）
  3. 我们尝试了什么（展开 10 条负结果，每条带数据）
  4. 这些边界判断告诉我们什么（n=600 + 类不平衡的真实约束）
  5. 鲁棒性 + 错误分析 + 深度学习对照
- **强项**：诚实、罕见——大多数学生只讲成功
- **弱项**：可能让老师以为"你们工作很失败"——需要在叙事开头先说成功是什么
